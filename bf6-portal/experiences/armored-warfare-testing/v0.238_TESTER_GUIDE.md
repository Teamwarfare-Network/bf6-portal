# TWL Tanks Sandbox - Test Checklist

Printable checklist to cover script + strings coverage with minimal steps.

Assumptions:
- Local test server with 2+ players (or bots).
- Map: Badlands (Blackwell Fields) unless noted.
- Admin = player ID 0.
- Access to Ready dialog (triple-tap interact) and Admin Panel.


## 1) Boot + Map Detection + Base State
- [ ] Start experience fresh.
- [ ] HUD branding shows current version.
- [ ] Ready dialog map label shows Blackwell Fields.
- Expected strings:
  - [ ] `twl.hud.branding.title`
  - [ ] `twl.hud.branding.subtitle`
  - [ ] `twl.readyDialog.mapLabel`
  - [ ] `twl.maps.blackwellFields`
  - [ ] `twl.hud.roundText`
  - [ ] `twl.hud.roundStateNotReady`
  - [ ] `twl.hud.helpText`

## 2) HUD Integrity (Baseline)
- [ ] Verify Round counter shows 1 / Max rounds.
- [ ] Verify Round Kills are 0-0.
- [ ] Verify Total Kills are 0-0.
- [ ] Verify Round Wins are 0-0.
- [ ] Verify Ties are 0 (Victory dialog later).

## 2) Ready Dialog Open + Roster Basics
- [ ] Triple-tap interact to open Ready dialog.
- [ ] Header, roster, and labels render.
- Expected strings:
  - [ ] `twl.readyDialog.header`
  - [ ] `twl.readyDialog.playerNameFormat`
  - [ ] `twl.readyDialog.status.ready`
  - [ ] `twl.readyDialog.status.notReady`
  - [ ] `twl.readyDialog.baseStatus.in`
  - [ ] `twl.readyDialog.baseStatus.out`
  - [ ] `twl.readyDialog.buttons.ready`
  - [ ] `twl.readyDialog.buttons.notReady`
  - [ ] `twl.readyDialog.buttons.swapTeams`

## 3) Matchup / Best-Of / Min Players / Target Kills
- [ ] Click Matchup +/-.
- [ ] Click Best-of +/-.
- Expected strings:
  - [ ] `twl.readyDialog.matchupFormat`
  - [ ] `twl.readyDialog.matchupChanged` (world log)
  - [ ] `twl.readyDialog.bestOfLabel`
  - [ ] `twl.readyDialog.bestOfChanged` (world log)
  - [ ] `twl.readyDialog.minPlayersToStartFormat`
  - [ ] `twl.readyDialog.targetKillsToWinFormat`
  - [ ] `twl.hud.roundLiveHelpFormat` (when round goes live)

## 3a) Matchup Spawn Behavior (Not Live)
- [ ] Set 1v1: confirm 1 vehicle per side spawns.
- [ ] Set 2v2: confirm 2 vehicles per side spawn.
- [ ] Set 3v3: confirm 3 vehicles per side spawn.
- [ ] Set 4v4: confirm 4 vehicles per side spawn.
- [ ] Downshift (4v4 -> 3v3): destroy the 4th slot vehicle and confirm it does not respawn.

## 4) Ready / Not Ready / In-Base Gating
- [ ] Toggle Ready on/off while in base.
- [ ] Step out of base and return.
- Expected strings:
  - [ ] `twl.notifications.enteredMainBase`
  - [ ] `twl.notifications.exitedMainBase`
  - [ ] `twl.notifications.readyupReturnToBaseNotLive`
  - [ ] `twl.overLine.title`
  - [ ] `twl.overLine.subtitle`

## 4a) Countdown Cancel Edge Cases
- [ ] Start countdown, then have any player unready.
- [ ] Start countdown, then have a player leave main base.
- [ ] Verify countdown UI hides and round does not start.
 - [ ] Start countdown, then use Admin "End Round" and confirm no late countdown ticks fire after the phase change.
 - [ ] Start countdown, then swap matchup preset and confirm no stale countdown restarts.

## 5) Start Round (All Ready)
- [ ] All players Ready in base until countdown starts.
- [ ] Observe countdown and GO.
- Expected strings:
  - [ ] `twl.countdown.format`
  - [ ] `twl.countdown.go`
  - [ ] `twl.hud.roundStateLive`
  - [ ] `twl.notifications.roundStarted`

## 6) Vehicle Spawn / Registration / Ownership
- [ ] Enter vehicles for each team.
- [ ] Destroy a vehicle while round is LIVE.
- Expected strings:
  - [ ] `twl.messages.init`
  - [ ] `twl.messages.vehicleSpawned`
  - [ ] `twl.messages.vehicleRegisteredNew`
  - [ ] `twl.messages.vehicleReturned`
  - [ ] `twl.messages.vehicleReRegistered`
  - [ ] `twl.messages.vehicleStolenRegister` (enter enemy vehicle)
 - [ ] Verify each vehicle is registered to exactly one team at a time (no double registration messages).
 - [ ] Verify no scoring or registration changes occur when round is NOT LIVE (pre-round).

## 7) Round Scoring
- [ ] Kill a registered enemy vehicle during LIVE.
- [ ] Verify round kills HUD and Round Wins update.
- Expected strings:
  - [ ] `twl.hud.labels.roundKills`
  - [ ] `twl.hud.labels.roundWins`
  - [ ] `twl.notifications.roundEndWin`
  - [ ] `twl.notifications.roundEndTie` (force tie via Admin Panel)

## 7a) Wins / Losses / Ties Math
- [ ] Force a tie round using Admin Panel and verify both teams show +1 tie.
- [ ] Force a Team 1 win and verify Team 1 wins +1, Team 2 losses +1.
- [ ] Force a Team 2 win and verify Team 2 wins +1, Team 1 losses +1.
- [ ] Verify Round Wins HUD matches Victory dialog totals.

## 8) Round End / Redeploy
- [ ] End a round (kill target or Admin End Round).
- [ ] Observe round end dialog and redeploy timer.
- Expected strings:
  - [ ] `twl.roundEnd.roundNumber`
  - [ ] `twl.roundEnd.draw` (if tie)
  - [ ] `twl.notifications.roundOverRedeploying`
 - [ ] While redeploy timer is active, verify no round-start countdown resumes or duplicates.

## 9) Victory Dialog
- [ ] End a match (best-of wins or Admin Panel).
- Expected strings:
  - [ ] `twl.victory.wins`
  - [ ] `twl.victory.loses`
  - [ ] `twl.victory.draws`
  - [ ] `twl.victory.teamOutcomeFormat`
  - [ ] `twl.victory.roundWinsFormat`
  - [ ] `twl.victory.roundLossesFormat`
  - [ ] `twl.victory.roundTiesFormat`
  - [ ] `twl.victory.totalKillsFormat`
  - [ ] `twl.victory.roundsSummaryFormat`
  - [ ] `twl.victory.restartInFormat`
  - [ ] `twl.victory.screenshotPrompt`
  - [ ] `twl.victory.totalMatchTimeLabel`

## 9a) Admin Audit on Victory
- [ ] Use Admin Panel at least once during the match.
- [ ] Verify Victory dialog shows admin action count.

## 10) Admin Panel Coverage
- [ ] Open Admin Panel button on Ready dialog.
- [ ] Press each Admin action button once.
- Expected strings:
  - [ ] `twl.adminPanel.buttons.panel`
  - [ ] `twl.adminPanel.accessed`
  - [ ] `twl.adminPanel.actionPressed`
  - [ ] `twl.adminPanel.actionCountFormat`
  - [ ] `twl.adminPanel.actionCountVictoryFormat`
  - [ ] All `twl.adminPanel.actions.*`
  - [ ] All `twl.adminPanel.labels.*`
  - [ ] `twl.adminPanel.tester.header`
  - [ ] All `twl.adminPanel.tester.buttons.*`
  - [ ] All `twl.adminPanel.tester.labels.*`

## 11) Team Switching (Single Toggle)
- [ ] Use "Change Teams" in Ready dialog.
- Expected strings:
  - [ ] `twl.notifications.teamSwitch`
  - [ ] `twl.teamSwitch.buttons.cancel`
  - [ ] `twl.teamSwitch.debugTimeLimit` (if visible)
 - [ ] Verify team switch does not mutate scores, round phase, or vehicle registration.

## 12) Base Ammo Restock
- [ ] Enter and exit base to trigger restock.
- Expected strings:
  - [ ] `twl.notifications.ammoRestocked`

## 13) Debug Position
- [ ] Use Admin "Toggle Position Debug".
- Expected strings:
  - [ ] `twl.debug.adminPos`
  - [ ] `twl.debug.adminFacing`
  - [ ] `twl.system.genericCounter` (used in position widget values)
 - [ ] Verify debug tools do not change match-critical state (scores, phase, registration).

## 14) Vehicle Destroyed While NOT Live
- [ ] Destroy a registered vehicle before the round is live.
- Expected strings:
  - [ ] `twl.messages.vehicleDestroyedNotLive`
 - [ ] Confirm no score/round counters change during NOT LIVE.

## 14a) Disconnect / Rejoin Edge Cases
- [ ] Disconnect a ready player during NOT LIVE and confirm roster updates.
- [ ] Rejoin and confirm they are NOT READY by default.
- [ ] Disconnect during countdown and confirm countdown cancels.
 - [ ] Rejoin during countdown and confirm the countdown does not resume incorrectly.

## 15) Map Label Coverage
- [ ] Repeat Ready dialog map label check on each map in `MAP_CONFIGS`.
- Expected strings:
  - [ ] All `twl.maps.*` entries used in the script

## 16) String Formatting Sanity
- [ ] Verify formatted messages render values (no raw `{}` or `{0}` visible in HUD/logs).
- [ ] Trigger the multi-click detector and confirm the string resolves (if enabled).

## Notes
- Use Admin Panel to fast-forward: set wins, kills, and end rounds/match quickly.
- Use team swap to trigger stolen/returned messages if needed.
